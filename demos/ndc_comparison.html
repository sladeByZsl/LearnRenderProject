NDC 坐标系介绍

NDC（Normalized Device Coordinates，归一化设备坐标）是顶点着色器输出后、光栅化之前的坐标空间。三套主流图形 API 在这个阶段的坐标系定义不完全相同。

OpenGL
也适用于 OpenGL ES / WebGL
X -1 左边缘 → +1 右边缘
Y -1 下边缘 → +1 上边缘
朝上，符合数学习惯
Z -1 近裁剪面 → +1 远裁剪面
范围 [-1,+1]，对称但精度较低
Vulkan
现代低开销图形 API
X -1 左边缘 → +1 右边缘
Y -1 上边缘 → +1 下边缘
朝下！与 OpenGL/DX 相反
Z 0 近裁剪面 → +1 远裁剪面
范围 [0,+1]，近处精度更好
DirectX
D3D11 / D3D12，仅限 Windows
X -1 左边缘 → +1 右边缘
Y -1 下边缘 → +1 上边缘
朝上，与 OpenGL 一致
Z 0 近裁剪面 → +1 远裁剪面
范围 [0,+1]，近处精度更好

差异汇总

轴 OpenGL Vulkan DirectX 说明
X -1 左 → +1 右 -1 左 → +1 右 -1 左 → +1 右 三家完全一致，无差异
Y -1 下 → +1 上 -1 上 → +1 下 ⚠ -1 下 → +1 上 Vulkan Y 轴翻转，同一段代码渲染结果上下颠倒
Z [-1, +1] [0, +1] [0, +1] OpenGL 用对称范围（历史遗留），Vulkan/DX 改为 [0,1] 精度更好

现阶段学 OpenGL 不需要关心跨 API 差异。投影矩阵库（如 glm）会帮你处理 Z 范围的转换；Vulkan 的 Y 轴翻转通常通过扩展 VK_KHR_maintenance1 或在投影矩阵里手动翻转解决。
